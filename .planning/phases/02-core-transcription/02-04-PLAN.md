**Test steps:**

1. Launch the app: `python main.py`
   - Expected: Window opens with ttkbootstrap flatly theme, TranscriptionPanel visible (toolbar, empty queue, action bar, log panel)
   - Expected: Log panel shows device detection message (e.g., "Zařízení: CUDA -- [GPU name]" or "GPU nedostupné, používám CPU")

2. Click "Přidat soubory" and select a test audio file
   - Expected: File appears in Treeview with filename, size (e.g., "1.4 MB"), duration loading then showing (e.g., "1:32"), status "čeká"

3. Hover over the filename in the Treeview
   - Expected: Tooltip shows the full file path

4. Leave the output folder empty (default: same as source file)

5. Click "Přepsat"
   - Expected: Button changes to "Zastavit" (red)
   - Expected: Status changes to "zpracovává se..." with segment progress
   - Expected: Log shows VAD result, Whisper progress lines, completion message
   - Expected: Progress bar advances
   - Expected: UI remains responsive (you can scroll log, resize window)

6. Wait for completion
   - Expected: Status changes to "hotovo"
   - Expected: Log shows "Hotovo: [filename] -> [output path]"
   - Expected: `_prepis.txt` file saved next to the source audio file
   - Expected: Button reverts to "Přepsat" (green)

7. Open the `_prepis.txt` file -- verify it contains Czech text from the audio

8. Right-click on the "hotovo" row
   - Expected: Context menu with "Uložit jako..." appears

9. (Optional) Test error handling: add a non-audio file (e.g., .txt) and try to transcribe
   - Expected: Error appears in log and status shows "chyba" in red

10. (Optional) Test stop: add multiple files, start transcription, click "Zastavit"
    - Expected: Current file completes, remaining files revert to "čeká"
