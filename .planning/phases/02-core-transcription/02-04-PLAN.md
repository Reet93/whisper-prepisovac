  <how-to-verify>
**Test steps:**

1. Launch the app: `python main.py`
   - Expected: Window opens with ttkbootstrap flatly theme, TranscriptionPanel visible (toolbar, empty queue, action bar, log panel)
   - Expected: Log panel shows device detection message (e.g., "Zarizeni: CUDA -- [GPU name]" or "GPU nedostupne, pouzivam CPU")

2. **GPU detection log (SC-6 -- REQUIRED):** Confirm the log panel contains a device line. It should be one of:
   - "Zarizeni: CUDA -- [GPU name]" if NVIDIA GPU present
   - "Zarizeni: Apple MPS (Apple Silicon)" if on Apple Silicon Mac
   - "GPU nedostupne, pouzivam CPU" if no GPU
   This step is NOT optional. If the device line is missing, report as a failure.

3. Click "Pridat soubory" and select a test audio file
   - Expected: File appears in Treeview with filename, size (e.g., "1.4 MB"), duration loading then showing (e.g., "1:32"), status "ceka"

4. Hover over the filename in the Treeview
   - Expected: Tooltip shows the full file path

5. Leave the output folder empty (default: same as source file)

6. Click "Prepsat"
   - Expected: Button changes to "Zastavit" (red)
   - Expected: Status changes to "zpracovava se..." with segment progress
   - Expected: Log shows VAD result, Whisper progress lines, completion message
   - Expected: Progress bar advances
   - Expected: UI remains responsive (you can scroll log, resize window)

7. Wait for completion
   - Expected: Status changes to "hotovo"
   - Expected: Log shows "Hotovo: [filename] -> [output path]"
   - Expected: `_prepis.txt` file saved next to the source audio file
   - Expected: Button reverts to "Prepsat" (green)

8. Open the `_prepis.txt` file -- verify it contains Czech text from the audio

9. Right-click on the "hotovo" row
   - Expected: Context menu with "Ulozit jako..." appears

10. **Error handling (SC-5 -- REQUIRED):** Test error handling by adding a non-audio file (e.g., rename a .txt to .mp3) and try to transcribe it.
    - Expected: Error appears in log with descriptive message (e.g., "Chyba: [filename] -- [error details]")
    - Expected: Status shows "chyba" in red text in Treeview
    - Expected: App does NOT crash -- other files can still be transcribed
    This step is NOT optional. If the app crashes or shows no error message, report as a failure.

11. (Optional) Test stop: add multiple files, start transcription, click "Zastavit"
    - Expected: Current file completes, remaining files revert to "ceka"
