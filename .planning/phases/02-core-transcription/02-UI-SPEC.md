# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Core Transcription.
> Scope: file queue Treeview, toolbar, action bar, real-time log panel, GPU detection feedback, error states, output save behavior.
> Phase 1 window skeleton (header/footer/language switcher) is locked — this contract covers only content_frame additions.
>
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | ttkbootstrap 1.20.2 |
| Preset | `flatly` theme (inherited from Phase 1 — locked) |
| Component library | ttkbootstrap built-ins: `ttk.Treeview`, `ttk.Button`, `ttk.Entry`, `ttk.Label`, `ttk.Progressbar`, `ttkbootstrap.ScrolledText`, `ttkbootstrap.ToolTip` |

**shadcn gate:** Not applicable. Python/Tkinter project. shadcn requires a JavaScript/React stack.

**Inheritance note:** All Phase 1 design tokens (spacing, typography, color, layout manager) are inherited without change. This contract extends, never overrides, Phase 1 decisions.

## Spacing Scale

Inherited from Phase 1 (source: `01-UI-SPEC.md`). No exceptions added for Phase 2.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between adjacent labels |
| sm | 8px | Padding inside compact controls, Treeview cell internal padding |
| md | 16px | Default widget padding (`padx=16, pady=16`), action bar internal padding |
| lg | 24px | Section padding, gap between Treeview and action bar, gap between action bar and log |
| xl | 32px | Layout gaps between major frame sections |
| 2xl | 48px | Major section breaks (not used in Phase 2 content area — panels are tight) |
| 3xl | 64px | Page-level spacing (not used) |

**Exceptions for Phase 2:**
- Treeview minimum height: 6 rows visible at default window size (720x480). Row height controlled by ttkbootstrap's `flatly` style — do not override row height manually.
- Log panel minimum height: 120px to show at least 5 log lines at 10pt with 1.4 line-height.
- Overall progress bar height: 12px (`ttk.Progressbar` default length; do not override height — use default).
- "Přepsat" / "Zastavit" button minimum width: 120px (fits longest label "Zastavit" at 10pt + padding).
- Output path Entry field: expands horizontally to fill remaining action bar width (weight=1 in grid).
- Window default size: remains 720x480 from Phase 1. Content_frame now holds three panels stacked vertically — the window remains scrollable via the content itself, not a scrolled window.

## Typography

Inherited from Phase 1 (source: `01-UI-SPEC.md`). Phase 2 adds one new role for log text.

| Role | Size | Weight | Line Height | ttkbootstrap Style |
|------|------|--------|-------------|--------------------|
| Body | 10pt | normal (400) | 1.4 | `TLabel` default |
| Label | 9pt | normal (400) | 1.3 | `TLabel` with `font=("", 9)` |
| Heading | 11pt | bold (700) | 1.2 | `TLabel` with `font=("", 11, "bold")` |
| Title / Window | 13pt | bold (700) | 1.2 | `TLabel` with `font=("", 13, "bold")` |
| Log / Monospace | 9pt | normal (400) | 1.4 | `ScrolledText` with `font=("Consolas", 9)` on Windows, `font=("Menlo", 9)` on macOS |

**Log font rationale (Claude's Discretion):** Monospace is mandatory for the log panel. Log lines contain timestamps, filenames, and progress numbers that must align vertically for scannability. Consolas (Windows) and Menlo (macOS) are the platform monospace standards and are always available — no font installation required.

**Usage in Phase 2:**
- Toolbar section heading (if used): Heading role, 11pt bold
- Treeview column headers: Body role, 10pt normal (ttkbootstrap Treeview heading style)
- Treeview cell text: Body role, 10pt normal
- Status cell text for active row: Body role, 10pt bold (weight override for "zpracovává" state only)
- Action bar labels ("Výstupní složka:", progress counter): Label role, 9pt normal
- Log panel content: Log/Monospace role, 9pt Consolas/Menlo
- Button labels ("Přepsat", "Přidat soubory", etc.): Body role, 10pt normal (ttkbootstrap Button default)

**Constraint:** Do not mix more than 2 font weights in any single view. Phase 2 continues using normal (400) + bold (700) only.

Phase 2 populates `content_frame` (Phase 1's row=1 placeholder). The root window structure from Phase 1 is locked — do not modify `main_frame`, `header_frame`, or `footer_frame`.

```

  └── main_frame (ttk.Frame, padding=16)
        ├── header_frame (row=0) — LOCKED from Phase 1
        ├── content_frame (row=1, sticky="nsew", weight=1) — Phase 2 populates this
        │     └── TranscriptionPanel (ttk.Frame, fills content_frame)
        │           ├── toolbar_frame (row=0, sticky="ew")

        │           │     └── btn_remove       ("Odebrat vybrané", bootstyle="secondary")
        │           ├── queue_frame (row=1, sticky="nsew", weight=1)
        │           │     └── ttk.Treeview (columns: Soubor, Velikost, Délka, Stav)
        │           │           └── ttk.Scrollbar (vertical, packed right)
        │           ├── action_frame (row=2, sticky="ew")
        │           │     ├── lbl_output       ("Výstupní složka:", 9pt)
        │           │     ├── entry_output_dir (Entry, expands, weight=1)
        │           │     ├── btn_browse       ("...", 32px wide, bootstyle="secondary")
        │           │     ├── btn_transcribe   ("Přepsat" / "Zastavit", 120px min, bootstyle="success"/"danger")
        │           │     └── progressbar_overall (ttk.Progressbar, mode="determinate", 160px wide)
        │           └── log_frame (row=3, sticky="ew")
```

**Layout rules (Phase 2 additions):**
- `TranscriptionPanel` uses `grid` layout exclusively. No `pack` or `place`.
- `queue_frame` has `rowconfigure(0, weight=1)` and `columnconfigure(0, weight=1)` to allow the Treeview to expand when window is resized vertically.
- `action_frame` uses a single row of widgets in a grid. `entry_output_dir` column has `weight=1` so it stretches.
- `log_frame` is fixed-height (120px minimum) at the bottom. It does not expand on resize — `queue_frame` takes all extra vertical space.
- `toolbar_frame` buttons are left-aligned with `sm` (8px) gap between them.

## Treeview Contract

**Columns:**

| Column ID | Header (Czech) | Header (English) | Min Width | Stretch | Anchor |
|-----------|---------------|-----------------|-----------|---------|--------|
| `filename` | `Soubor` | `File` | 200px | yes (weight=1) | w (left) |
| `filesize` | `Velikost` | `Size` | 70px | no | e (right) |
| `duration` | `Délka` | `Duration` | 70px | no | e (right) |
| `status` | `Stav` | `Status` | 100px | no | center |

**Row behavior:**
- Single-click selects a row. `extended` selectmode allows Shift+click and Ctrl+click for multi-select.
- "Odebrat vybrané" removes all selected rows. Button is disabled when transcription is running and the selected row has status "zpracovává" — guard prevents removal of the active file.
- "Uložit jako" is available via right-click context menu on rows with status "hotovo". It opens `tkinter.filedialog.asksaveasfilename` with `.txt` filter.
- Double-click on a "hotovo" row triggers "Uložit jako".
- Tooltip on `filename` cell shows the full file path. Implemented via `ttkbootstrap.ToolTip` bound to the Treeview widget with dynamic text update on `<Motion>` to detect which row is hovered.

**Status display:**

| Status | Czech label | English label | Tag applied |
|--------|------------|---------------|-------------|
| Waiting | `čeká` | `waiting` | `waiting` |
| Processing | `zpracovává se…` | `processing…` | `processing` |
| Done | `hotovo` | `done` | `done` |
| Error | `⚠ chyba` | `⚠ error` | `error` |

Note: The `⚠` character (U+26A0) is the warning prefix for error rows. It is a standard Unicode symbol with good cross-platform coverage on Windows 10+ and macOS. No icon library required.

**Per-file progress (Claude's Discretion — D-14):** The status cell of the active row shows `zpracovává se… N/T` (N segments completed out of T total) while processing. Format: `zpracovává se… 12/87`. This is updated from the progress queue without a separate progress bar column — keeps the table compact.

**Duration loading:** ffprobe is called asynchronously when a file is added to the queue. During the brief ffprobe call, the `Délka` cell shows `…` (ellipsis). On completion it shows `MM:SS` format (e.g., `14:32`). If ffprobe fails, shows `–` (en-dash, not a hyphen).

**Filesize format:** Human-readable with one decimal place. Examples: `1.4 MB`, `320 KB`, `12.7 MB`. Computed from `Path.stat().st_size` using standard IEC binary units (1 KB = 1024 bytes).

## Log Panel Contract

The log panel occupies `log_frame` (row=3 in TranscriptionPanel).

**Widget:** `ttkbootstrap.ScrolledText`, `state="disabled"` (read-only), `height=8` (8 text lines at 9pt), `font=("Consolas", 9)` on Windows / `font=("Menlo", 9)` on macOS. `wrap="word"`.

**Auto-scroll behavior:** On each new line appended, scroll to end via `log.see(tk.END)` — always shows the latest output. User can scroll up to review history; scrolling up does NOT disable auto-scroll (auto-scroll resumes on the next line).

**Log line format (Claude's Discretion):**

```
[HH:MM:SS] MESSAGE
```

Timestamp uses 24-hour local time (`datetime.now().strftime("%H:%M:%S")`). No date prefix — log session is per-app-launch.

**Log line templates:**

| Event | Czech template | Tag |
|-------|---------------|-----|
| App startup / device detected (GPU) | `[HH:MM:SS] Zařízení: CUDA – {GPU_NAME}` | `device` |
| App startup / device detected (CPU) | `[HH:MM:SS] GPU nedostupné, používám CPU` | `device` |
| App startup / device detected (MPS) | `[HH:MM:SS] Zařízení: Apple MPS (Apple Silicon)` | `device` |
| File added to queue | `[HH:MM:SS] Přidán soubor: {FILENAME}` | `info` |
| Batch started | `[HH:MM:SS] Zahajuji přepis {N} souborů…` | `info` |
| File processing started | `[HH:MM:SS] Zpracovávám: {FILENAME}` | `info` |
| VAD result | `[HH:MM:SS] VAD: {N} řečových segmentů nalezeno ({DURATION}s řeči z {TOTAL}s)` | `info` |
| Whisper progress | `[HH:MM:SS] → {N}/{T} segmentů` | `progress` |
| File done | `[HH:MM:SS] Hotovo: {FILENAME} → {OUTPUT_PATH}` | `done` |
| Batch done | `[HH:MM:SS] Přepis dokončen. {N} souborů hotovo.` | `done` |
| Stop requested | `[HH:MM:SS] Zastavuji po dokončení aktuálního souboru…` | `info` |
| Batch stopped | `[HH:MM:SS] Přepis zastaven. {DONE} hotovo, {REMAINING} čeká.` | `info` |
| Error | `[HH:MM:SS] Chyba: {FILENAME} — {ERROR_TYPE}: {ERROR_DETAIL}` | `error` |

**Whisper progress lines** use `progress` tag (muted gray `#95A5A6`) to visually de-emphasize the frequent segment counter updates, keeping completion and error lines prominent.

## GPU Detection Feedback Contract

GPU detection runs at app startup (before the first transcription). No blocking dialog. Behavior:

1. Detection result is written to log at startup with `device` tag (bold dark navy).
2. No visible badge, indicator, or persistent label is added to the UI for Phase 2. The log is the only persistent record of which device is active.
3. If GPU detection takes more than 2 seconds (slow PyTorch init), do not block the main thread — run in a background thread and log result when ready.
