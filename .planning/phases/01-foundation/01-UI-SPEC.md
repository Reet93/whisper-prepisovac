# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Scope: ttkbootstrap window skeleton, i18n infrastructure, language switcher widget, resource path abstraction.
> No transcription UI in this phase — the contract covers only the base window and language switching.
>
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | ttkbootstrap 1.20.2 |
| Preset | `flatly` theme (light, professional, Bootstrap-inspired) |
| Component library | ttkbootstrap built-ins (`ttk.Frame`, `ttk.Label`, `ttk.Combobox`, `ttk.Button`) |
| Icon library | none (Phase 1 has no icons — text labels only) |
| Font | System default via ttkbootstrap: Segoe UI on Windows, SF Pro on macOS |

**Theme rationale:** `flatly` is ttkbootstrap's clean light theme with muted primary color (#2C3E50 navy/dark, #18BC9C teal accent). It reads as a professional desktop tool, avoids harsh contrast, and is visually consistent across Windows and macOS without platform-specific hacks. Dark mode support (ttkbootstrap `darkly`) is deferred to v2.

**shadcn gate:** Not applicable. This is a Python/Tkinter project. shadcn requires a JavaScript/React stack.

## Spacing Scale

Declared values (all multiples of 4 pixels — Tkinter `padding` uses integer px):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between adjacent labels |
| sm | 8px | Padding inside compact controls (Combobox internal) |
| md | 16px | Default widget padding (`padx=16, pady=16`) |
| lg | 24px | Section padding, outer frame margins |
| xl | 32px | Layout gaps between major frame sections |
| 2xl | 48px | Major section breaks (used in Phase 2+ for panel separation) |
| 3xl | 64px | Page-level spacing (not used in Phase 1) |

Exceptions:
- Window minimum size: 480px wide x 320px tall — smallest sensible desktop window for the Phase 1 skeleton
- Language switcher Combobox: minimum 120px wide (fits longest option "English" with padding)
- Title bar height: controlled by OS — not overridable in Tkinter

## Typography

ttkbootstrap maps font sizes through ttk style definitions. Sizes are declared in points (pt), which scale correctly with OS DPI settings on both Windows and macOS.

| Role | Size | Weight | Line Height | ttkbootstrap Style |
|------|------|--------|-------------|--------------------|
| Body | 10pt | normal (400) | 1.4 | `TLabel` default |
| Label | 9pt | normal (400) | 1.3 | `TLabel` with `font=("", 9)` |
| Heading | 11pt | bold (700) | 1.2 | `TLabel` with `font=("", 11, "bold")` |
| Title / Window | 13pt | bold (700) | 1.2 | `TLabel` with `font=("", 13, "bold")` |

**Usage in Phase 1:**
- Window title bar text: OS-controlled, set via `root.title("Whisper Přepisovač")` — not styled by Tkinter
- In-window app heading (if shown): Title role, 13pt bold
- Language switcher label "Jazyk / Language:": Label role, 9pt normal
- Placeholder body text (if shown): Body role, 10pt normal

**Constraint:** Do not mix more than 2 font weights in any single view. Phase 1 uses normal + bold only.

## Color

ttkbootstrap `flatly` theme provides the following semantic palette. These values are the actual hex colors from the flatly Bootstrap CSS specification:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (white) | Main window background, frame backgrounds |
| Secondary (30%) | `#F8F9FA` (near-white gray) | Section frames, inner panel backgrounds |
| Accent (10%) | `#18BC9C` (teal) | Active/selected state on language switcher, focus ring on interactive elements |
| Text primary | `#2C3E50` (dark navy) | All body text, labels, headings |
| Text muted | `#95A5A6` (medium gray) | Placeholder text, disabled labels |
| Destructive | `#E74C3C` (Bootstrap red) | Destructive actions only — not used in Phase 1 |
| Border | `#DEE2E6` (light gray) | Widget borders, separator lines |

Accent (`#18BC9C` teal) is reserved for:
1. The currently selected item in the language Combobox (selection highlight)
2. Focus ring on keyboard-focused interactive widget
3. Phase 2+: Primary action buttons ("Přepsat", "Přepsat + Upravit")

Accent is NOT used for: decorative borders, background fills, text color on non-interactive elements.

**Implementation note:** ttkbootstrap `flatly` applies these colors automatically via its ttk style engine. Do not override theme colors manually — use `bootstyle` parameter on ttkbootstrap widgets (e.g., `bootstyle="success"` for accent, `bootstyle="danger"` for destructive). Direct hex overrides break cross-platform consistency.

## Copywriting Contract

Phase 1 scope is a skeleton window with a language switcher. All strings must pass through the `_()` i18n function — these are the canonical source strings in Czech (primary language) with English translations.

| Element | Czech (primary) | English (secondary) | Notes |
|---------|----------------|---------------------|-------|
| Window title | `Whisper Přepisovač` | `Whisper Transcriber` | Set via `root.title(_("app.title"))` |
| App heading (in-window) | `Whisper Přepisovač` | `Whisper Transcriber` | Displayed as Title role heading if shown |
| Language switcher label | `Jazyk:` | `Language:` | Precedes the Combobox |
| Language option: Czech | `Čeština` | `Czech` | Value in Combobox for `cs` locale |
| Language option: English | `English` | `English` | Value in Combobox for `en` locale — same in both languages |
| Placeholder body (if used) | `Vítejte. Funkce přepisu budou přidány v dalších fázích.` | `Welcome. Transcription features will be added in later phases.` | Optional skeleton placeholder text |
| Language restart notice (if language switch requires restart) | `Jazyk byl změněn. Restartujte aplikaci pro použití změn.` | `Language changed. Restart the app to apply changes.` | Shown in status bar or modal if restart is required |

**Primary CTA:** None in Phase 1. The language Combobox is the only interactive element.

**Empty state:** Not applicable to Phase 1 (no data-driven views).

**Error state:** If locale `.mo` file fails to load, fall back silently to Czech (fallback=True in `gettext.translation()`). Do not show an error dialog for locale failure in Phase 1 — log it to stderr only.

**Destructive actions:** None in Phase 1.

**i18n key format:** Use dot-notation keys as msgid in `.po` files (e.g., `app.title`, `ui.language_label`, `ui.language_cs`, `ui.language_en`). This avoids collisions and makes Poedit navigation easier than using raw Czech sentences as keys.

## Window Layout Contract

Phase 1 establishes the root window structure that all later phases build on. This layout is locked — Phase 2 must not restructure the root frame hierarchy.

```
root (ttkbootstrap.Window, theme="flatly")
  └── main_frame (ttk.Frame, padding=16)
        ├── header_frame (ttk.Frame)
        │     └── app_title_label (ttk.Label, Title role, row=0)
        ├── content_frame (ttk.Frame, expand=True, fill=BOTH)
        │     └── [placeholder label in Phase 1, replaced in Phase 2]
        └── footer_frame (ttk.Frame)
              ├── language_label (ttk.Label, Label role)
              └── language_combobox (ttk.Combobox, width=12, state="readonly")
```

**Layout rules:**
- `main_frame` uses `grid` layout manager. Do not mix `pack` and `grid` in the same parent.
- `content_frame` takes `row=1, sticky="nsew"` with `columnconfigure(0, weight=1)` and `rowconfigure(1, weight=1)` on `main_frame` — this makes content expand to fill remaining space when the window is resized.
- `footer_frame` is pinned to the bottom via `row=2, sticky="ew"`.
- Language switcher lives in `footer_frame` right-aligned (`sticky="e"`).
- Window minimum size: `root.minsize(480, 320)`.
- Window default size: `root.geometry("720x480")`.

## i18n Infrastructure Contract

This section defines the file layout and loading pattern that must exist before any widget is created (per requirement UI-02).

**File structure:**
```
locale/
  cs_CZ/
    LC_MESSAGES/
      messages.po
      messages.mo
  en_US/
    LC_MESSAGES/
      messages.po
      messages.mo
```

**Loading pattern** (must execute before `root = ttkbootstrap.Window(...)` is called):
```python
import gettext
from pathlib import Path

def get_locale_dir() -> Path:
    # Uses get_resource_path() from Phase 1 resource helper
    return get_resource_path("locale")

def set_language(lang_code: str) -> None:
    """Install _() into builtins for the selected language. lang_code: 'cs' or 'en'."""
    locale_map = {"cs": "cs_CZ", "en": "en_US"}
    gettext.translation(
        "messages",
        localedir=str(get_locale_dir()),
        languages=[locale_map.get(lang_code, "cs_CZ")],
        fallback=True,
    ).install()

# Call before any widget code:
set_language("cs")  # default Czech
```

**Language switch behavior:** Switching language in Phase 1 may require restart (restart notice string defined above). Live in-place widget text update without restart is a Phase 3 enhancement (settings persistence). Phase 1 just needs the mechanism to work on launch.

## Resource Path Contract

`get_resource_path()` must be defined in a standalone module (`utils/resource_path.py` or similar) and imported by all modules that reference bundled assets.

```python
import sys
from pathlib import Path

def get_resource_path(relative_path: str) -> Path:
    """Resolve path to a bundled resource.

    Works in both development (relative to project root) and PyInstaller
    frozen context (relative to sys._MEIPASS).
    """
    if getattr(sys, "frozen", False):
        base = Path(sys._MEIPASS)
    else:
        # Development: resolve relative to this file's location
        base = Path(__file__).parent.parent
    return base / relative_path
```

This function is the only approved way to reference `locale/`, `models/`, and `bin/` paths throughout the entire codebase.
