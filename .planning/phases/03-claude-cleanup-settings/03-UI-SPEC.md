# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Claude Cleanup + Settings.
> Scope: "Přepsat + Upravit" pipeline, standalone "Upravit" button, API key banner and management,
> settings modal (General + Claude tabs), inline prompt editor, context profiles dropdown,
> cost estimate display, VAD spinner, live language reload, file collision handling.
>
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | ttkbootstrap 1.20.2 |
| Preset | `flatly` theme — inherited from Phase 1 (approved 2026-03-28) |
| Component library | ttkbootstrap built-ins (`ttk.Frame`, `ttk.Label`, `ttk.Button`, `ttk.Combobox`, `ttk.Entry`, `ttk.Notebook`, `ttk.Scrollbar`, `ScrolledText`, `ToolTip`). Modal dialogs via `ttk.Toplevel`. |
| Icon library | None. The gear icon in the header is a Unicode character `\u2699` (⚙) rendered as a `ttk.Button` with symbol text, consistent with Phase 2 approach of text-only controls. |
| Font | System default via ttkbootstrap: Segoe UI on Windows, SF Pro on macOS. Log panel: monospace (`Consolas 9pt` on Windows, `Menlo 9pt` on macOS) — matches Phase 2 established pattern. |

**Theme continuation:** All new widgets use `bootstyle=` parameter on ttkbootstrap widgets. No direct hex color overrides. Dark mode deferred to v2.

**shadcn gate:** Not applicable. This is a Python/Tkinter project.

## Color

Inherited from Phase 1/2 palette. Phase 3 extends with two additional semantic log tags and one new status tag.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (white) | Main window, settings modal background, prompt editor background |
| Secondary (30%) | `#F8F9FA` (near-white gray) | Section frames inside settings tabs; API key banner background |
| Accent (10%) | `#18BC9C` (teal) | "Přepsat + Upravit" button (`bootstyle="success"`); "Přepsat" button when Claude key is active; active tab indicator in settings Notebook; cost estimate check indicator |
| Text primary | `#2C3E50` (dark navy) | All body text, labels, headings |
| Text muted | `#95A5A6` (medium gray) | Placeholder text in context field; cost estimate pre-send label; "Chunk 2/5" progress text |
| Destructive | `#E74C3C` (Bootstrap red) | "Odstranit API klíč" button (`bootstyle="danger"`); "chyba (Claude)" status tag in Treeview; error log tag |
| Warning | `#F39C12` (amber) | API key banner background tint when reappearing due to invalid key; `bootstyle="warning"` on banner label |
| Border | `#DEE2E6` (light gray) | Settings modal separator lines; inline prompt editor border |

**Accent (`#18BC9C` teal) is reserved for:**
1. "Přepsat + Upravit" primary action button
2. "Přepsat" button (both use `bootstyle="success"`)
3. Active selection state in language Combobox and context profile dropdown
4. Focus ring on keyboard-focused interactive widget
5. Settings Save button (`bootstyle="success"`)
6. `done` / `claude_done` status tags in Treeview and log

**Accent is NOT used for:** decorative borders, banner backgrounds, section dividers, informational icons.

**New Treeview status tag — `claude_error`:**
```python
self.tree.tag_configure("claude_error", foreground="#E74C3C", background="#FFF5F5")
```
Status text: `chyba (Claude)` / `error (Claude)` — rendered identically to the existing `error` tag visually, distinguished by label text only.

**New log tags (Phase 3 additions to existing log panel):**
```python
self.log.text.tag_configure("claude",    foreground="#2C3E50")         # Claude API progress
self.log.text.tag_configure("claude_done", foreground="#18BC9C")       # Claude completed
self.log.text.tag_configure("cost",      foreground="#95A5A6")         # Token/cost lines
```

**VAD spinner:** Animated dots (`·`, `··`, `···`) appended to the status cell text during VAD preprocessing. No new color — uses existing `processing` tag foreground `#F39C12`.

## Interaction Contract

### Settings Modal

- Opens on General tab always (D-16 from CONTEXT.md)
- Escape key closes without saving
- Save validates output folder existence before closing; if invalid, shows error label inline (no dialog)
- API key field: masked with `*` by default; toggle button reveals plaintext. On "Uložit" with a new key value, fires a background validation call — shows "Ověřuji..." label, then "Klíč platný ✓" or error. Dialog does NOT close until validation resolves.
- `grab_set()` on the dialog — main window is non-interactive while settings is open

### Context Profiles Dropdown

- Default option: "Žádný profil" (no free-text context sent to Claude)
- Profile management via `btn_manage_profile` ("...") which opens a `tk.Menu` with: "Nový profil...", "Přejmenovat...", "Smazat"
- "Nový profil..." opens a simple `simpledialog.askstring` for the name, then saves current context text as the profile content
- Profile switch updates `entry_context` with saved profile text

### Inline Prompt Editor

- Collapsed by default; `btn_toggle_prompt` toggles visibility
- Reset button restores content of `transcript-processor-prompt.md` for the current language
- Cost estimate label updates when prompt text changes (debounced 500ms after last keypress, based on character count approximation)
- Prompt edits persist to settings JSON on save or when "Přepsat + Upravit" is clicked

### "Přepsat + Upravit" Button States

| Condition | State | Visual |
|-----------|-------|--------|
| No files in queue | disabled | gray, no tooltip |
| Files in queue, no API key | disabled | gray, tooltip: "Nejdříve nastavte API klíč Claude" |
| Files in queue, API key stored | enabled | teal (`bootstyle="success"`) |
| Cleanup in progress | disabled | gray (pipeline running, cannot re-trigger) |

### "Upravit" Standalone Button States

| Condition | State |
|-----------|-------|
| No `done` rows in queue | disabled |
| ≥1 `done` row exists (any, not just selected) | enabled |
| Cleanup in progress | disabled |

### VAD Spinner

During VAD preprocessing, the status column of the active row cycles through `·`, `··`, `···` using `root.after(400, ...)` tick. Once VAD completes, the cell transitions to the normal `zpracovávám` + `%` progress pattern.

### Live Language Reload (Phase 3 feature)

Switching language in the footer Combobox calls `set_language()` then rebuilds all widget text via a `reload_ui_strings()` method on `MainWindow` (and delegates to `TranscriptionPanel.reload_strings()` and the settings dialog if open). No restart required. Default prompt file path switches to `transcript-processor-prompt-{lang}.md`.
